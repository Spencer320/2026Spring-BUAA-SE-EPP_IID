### 3.3 API 模块设计

用户端 API（`researchAgentApi.js`）：

```js
const PREFIX = 'research-agent'

export function listSessions(params)                 // GET  /api/research-agent/sessions/
export function createSession(data)                 // POST /api/research-agent/sessions/
export function getSession(sessionId)               // GET  /api/research-agent/sessions/:id/
export function postMessage(sessionId, content)     // POST /api/research-agent/sessions/:id/messages/
export function getTask(taskId)                     // GET  /api/research-agent/tasks/:id/
export function postIntervention(taskId, body)      // POST /api/research-agent/tasks/:id/intervention/
export function postCancelTask(taskId)              // POST /api/research-agent/tasks/:id/cancel/
```

### 3.4 核心交互与状态流转

#### 3.4.1 会话主流程

```
创建会话
  ↓
发送用户指令（postMessage）
  ↓
后端创建 AgentTask(status=pending) 并启动编排线程
  ↓
用户端轮询 getTask
  ↓
running → waiting_user（出现 intervention）
  ↓
用户选择 approve / reject / revise
  ↓
终态：completed / failed / cancelled
```

#### 3.4.2 任务状态机

| 状态             | 含义           | 下一状态                                      |
| ---------------- | -------------- | --------------------------------------------- |
| `pending`      | 待执行         | `running`                                   |
| `running`      | 执行中         | `waiting_user` / `completed` / `failed` |
| `waiting_user` | 等待干预       | `running` / `cancelled`                   |
| `completed`    | 完成           | 终态                                          |
| `failed`       | 执行失败       | 终态                                          |
| `cancelled`    | 用户拒绝或取消 | 终态                                          |

#### 3.4.3 编排步骤与行为日志联动

- 编排器 `_append_step(...)` 在写步骤 JSON 的同时自动调用 `_append_behavior_log(...)`。
- `act` 阶段会附带 `operation_type=outbound_get`、`target_url`、`response_status`。
- 当 `response_status >= 400` 时强制标记 `is_exception=true`，用于后续审计筛选。

## 4. 功能二：科研助手行为审计（管理端）

### 4.1 数据模型设计

模型：`AgentBehaviorAuditLog`（`research_agent/models.py`）

| 字段                  | 类型          | 说明                                       |
| --------------------- | ------------- | ------------------------------------------ |
| `id`                | BigAutoField  | 主键                                       |
| `task`              | FK(AgentTask) | 关联任务                                   |
| `operation_type`    | Char(64)      | 操作类型，如 `outbound_get`/`navigate` |
| `target_url`        | Char(1024)    | 访问 URL                                   |
| `target_domain`     | Char(255)     | 访问域名（索引）                           |
| `request_headers`   | JSON          | 请求头快照                                 |
| `request_payload`   | JSON          | 请求体快照                                 |
| `action_payload`    | JSON          | 动作信息（如点击参数）                     |
| `response_status`   | Integer?      | HTTP 状态码                                |
| `is_exception`      | Bool          | 是否异常                                   |
| `exception_message` | Char(512)     | 异常说明                                   |
| `trace_detail`      | Text          | 轨迹描述                                   |
| `occurred_at`       | DateTime      | 行为发生时间（索引）                       |
| `created_at`        | DateTime      | 记录创建时间                               |

### 4.2 后端接口设计

#### 4.2.1 用户态探针上报

`POST /api/research-agent/tasks/<task_id>/behavior-logs/`

- 鉴权：用户 JWT
- 作用：科研助手执行引擎探针主动上报行为
- 必填：`operation_type`
- 自动处理：
  - 若未传 `target_domain`，由 `target_url` 自动解析
  - 若 `response_status >= 400`，自动判定 `is_exception=true`

#### 4.2.2 管理端日志列表

`GET /api/research-agent/manage/behavior-logs/`

支持筛选参数：

- `user_id`
- `task_id`
- `target_domain`
- `operation_type`
- `exception_status`（`all|true|false`）
- `date_from` / `date_to`
- `task_status`
- `page_num` / `page_size`

#### 4.2.3 单任务链路详情

`GET /api/research-agent/manage/tasks/<task_id>/behavior-chain/`

返回结构：

- `task`：任务元信息（task_id/session_id/user_id/status/step_seq）
- `logs`：按时间正序返回的完整行为链路

#### 4.2.4 结构化导出

`POST /api/research-agent/manage/behavior-logs/export/`

- 入参与列表筛选一致
- 导出上限：500 条
- 返回：`file_name` + Markdown `content` + `count`
- 文档内容包含：
  - 生成时间
  - 筛选条件回显
  - 表格化行为流水（时间/用户ID/任务ID/域名/操作/状态/异常/说明）

### 4.3 管理端页面设计

路由与菜单：

- 路由：`/research-agent-audit`
- 菜单：管理端左侧新增“科研助手行为审计”

## 5. 数据库迁移与兼容性

迁移文件：`research_agent/migrations/0002_agentbehaviorauditlog.py`

兼容性说明：

- 为新增表，不修改既有字段，不影响现有 `ResearchSession/AgentTask` 数据。
- 接口均为追加，不改变已有用户端 API 返回结构。
- 审计筛选默认仅统计 `pending/running/waiting_user/completed` 任务日志，避免历史终态噪声。
