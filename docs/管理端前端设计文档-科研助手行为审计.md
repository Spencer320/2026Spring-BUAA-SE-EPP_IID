## 3. 管理端页面设计（当前代码）

### 3.1 路由与菜单

- 路由：`/research-agent-audit`
- 菜单：`LayoutContainer.vue` 左侧新增“科研助手行为审计”
- 导航高亮：`activeMenu` 对 `/research-agent-audit` 单独匹配

### 3.2 页面结构

```text
ResearchAgentBehaviorAudit.vue
├── 筛选栏
│   ├── userId / taskId / targetDomain
│   ├── operationType / exceptionStatus / dateRange
│   └── 查询 / 重置 / 导出结构化文档
├── 审计日志表格
│   ├── 时间、用户ID、任务ID、域名、操作类型、状态码
│   ├── 异常状态、异常说明
│   └── 操作列：查看链路
├── 分页器（page_num/page_size）
└── 链路抽屉
    ├── 任务元信息（task/session/user/status）
    └── 行为时间线（按发生时间升序）
```

### 3.3 前端参数映射

`ResearchAgentBehaviorAudit.vue` 到后端查询参数映射：

- `filters.userId -> user_id`
- `filters.taskId -> task_id`
- `filters.targetDomain -> target_domain`
- `filters.operationType -> operation_type`
- `filters.exceptionStatus -> exception_status`
- `filters.dateRange[0/1] -> date_from/date_to`
- `currentPage/pageSize -> page_num/page_size`

### 3.4 导出行为

- 导出时移除分页参数，仅提交筛选参数；
- 后端返回 Markdown 文本，前端通过 `Blob + a[download]` 触发下载；
- 默认导出文件名回退为 `research-assistant-audit.md`。

## 4. 后端联动与接口约束

### 4.1 接口清单（已接入）

- `GET /api/research-agent/manage/behavior-logs/`
- `GET /api/research-agent/manage/tasks/{task_id}/behavior-chain/`
- `POST /api/research-agent/manage/behavior-logs/export/`

### 4.2 返回结构（当前代码事实）

日志列表：

```json
{
  "total": 123,
  "page_num": 1,
  "page_size": 20,
  "items": []
}
```

链路详情：

```json
{
  "task": {
    "task_id": "uuid",
    "session_id": "uuid",
    "user_id": "string",
    "status": "running|pending_action|completed|failed|cancelled",
    "step_seq": 0
  },
  "logs": []
}
```

导出：

```json
{
  "file_name": "research-assistant-audit-YYYYMMDD_HHMMSS.md",
  "content": "# 科研助手行为审计报告...",
  "count": 100
}
```

### 4.3 鉴权约束（当前风险点）

`research_agent/auth.py` 当前管理员鉴权优先依赖：

1. `X-Research-User-Id` + `X-Research-Role=admin` 请求头；或
2. 可被 `research_agent` 识别的 JWT。

而管理端前端默认仅发送 `Authorization`，未发送 `X-Research-*` 头，存在联调环境下 401/403 风险（见第 6 节 P0-1）。

## 6. 后续需要完善的代码工作

### P0（建议本迭代优先完成）

1. 管理端鉴权打通（避免审计接口 401/403）
- 现状：管理端页面只发 `Authorization`，`research_agent` 管理接口更依赖 `X-Research-*` 或特定 JWT 解析。
- 建议改动文件：
  - `EPP-Backend-Dev/research_agent/auth.py`
  - 或 `EPP-Frontend-Manager-Dev/src/utils/request.js`（按接口域补充请求头）
- 目标：管理员登录 token 能直接访问 `research-agent/manage/*`。

2. 操作类型筛选与真实日志值对齐
- 现状：前端下拉固定值为 `outbound_get/http_request/navigate/click/extract`，但编排器自动日志主值常为 `plan/decide/search/read/reflect/write`。
- 建议改动文件：
  - `EPP-Frontend-Manager-Dev/src/views/research_agent/ResearchAgentBehaviorAudit.vue`
- 目标：支持当前真实值，避免“筛选无结果”的假空态。

3. 减少重复请求
- 现状：`currentPage/pageSize` 的 `watch` 与 `handleSearch/handleReset` 同时触发，可能重复调用列表接口。
- 建议改动文件：
  - `EPP-Frontend-Manager-Dev/src/views/research_agent/ResearchAgentBehaviorAudit.vue`
- 目标：统一查询触发入口，避免一次操作触发多次请求。
