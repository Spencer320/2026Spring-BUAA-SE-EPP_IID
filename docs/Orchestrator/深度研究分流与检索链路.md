# 深度研究分流与检索链路

> 本文档描述当前实现（2026-05）。旧文档（如 `01-深度研究编排器.md`、`docs/history/04-*`）可能已过时，以本文件与代码为准。

## 1. 两个概念

| 概念 | 触发条件 | 编排 |
|------|----------|------|
| **深度研究功能** | `POST /api/research-agent/tasks/deep-research/`，`selected_papers` 为空 | `BasicOrchestratorRun`，`pipeline_mode=assistant`，扣 **科研助手** 配额 |
| **深度研究链路（综述）** | 同上 API，但 `selected_papers` 非空（展示区文献 id） | `AgentTask`，四阶段 `plan_decide → analyze → reflect → write`，`pipeline_mode=synthesis`，扣 **深度研究** 配额 |

前端不强制勾选文献；同一入口由后端分流。

## 2. HTTP 响应（202）

```json
{
  "task_id": "...",
  "session_id": "...",
  "status": "pending",
  "pipeline_mode": "assistant | synthesis",
  "orchestrator": "basic | deep_research"
}
```

轮询任务仍用 `GET /api/research-agent/tasks/<id>/`，`orchestrator` 字段与上表一致。

## 3. 综述链路：文献与检索

### 3.1 入口 enrich（`views._validate_selected_papers_from_shelf`）

每条 `selected_papers` 除标题、URL 外，还携带 `authors`、`abstract`、`search_query`、`file_extension`，写入 `runtime_config.selected_papers`。

### 3.2 种子证据（`research_agent/search_evidence.py`）

- `build_seed_citations(selected_papers)`：用户选定文献 → `source=user_selected` 的 citation，进入 `all_citations` / analyze 的 `raw_search_results`。
- `merge_citations(seed, external)`：去重合并，seed 优先。
- `is_effective_external_citation` / `count_effective_hits`：外搜有效性统计（不含 `user_selected`、`local_rag`）。

### 3.3 检索计划（plan_decide）

每个 `subtask` 必填 `search_queries`（1–4 条），每项含 `q`（短检索式）、`intent`、`rationale`。  
**不再**用 `subtask.goal` 直接调用 CrossRef。

执行：`_execute_subtask_searches` 对每条 `q` 调用 `_search_context`；结果写入 `search_execution_report` 并传入 analyze prompt。

### 3.4 检索审核（1c）

- 删除「WEB_SEARCH/ACADEMIC 失败 → local_rag 假成功」。
- 失败时 `citations=[]`，`degraded=true`，任务可继续（仅有 seed 时 analyze 须在 `search_notes` 说明外搜失败）。
- `result_payload.search_audit[]` 按子任务/轮次记录每条 query 的命中与降级。

### 3.5 Reflect 补搜（1d）

`reflect` JSON 扩展：

- `additional_search_queries`：`needs_optimization=yes` 时追加执行。
- `search_evidence_adequate`：`no` 时倾向继续优化。

| 项 | 位置 |
