# 快速开始

## 1. 加了什么、改了什么

- **编排（`orchestrator.py`）**  
  - 搜索阶段改为：**先跑真实检索**（工具/API），再把结果作为 `raw_search_results` 交给 Searcher LLM 做整理，减少「先编后搜」的幻觉。  
  - **引用链**：Reader / Reflector / Writer 贯通 `references`，失败兜底时尽量保留；最终 Markdown 增加「参考来源」区块。  
  - **延迟**：`read` / `reflect` / `write` 调用 LLM 时关闭 `enable_thinking`，减轻前端读超时。  
  - **深度**：默认最大反思/检索轮次提高到 **5**（仍受配置与上限约束）。

- **提示词（`prompts.py`）**  
  Searcher 必须基于真实检索结果、禁止编造 URL；Reader/Reflector/Writer 的 JSON 约定中加入 `references` 与正文引用约束。

- **执行器（已有能力，文档侧说明）**  
  学术 API（Semantic Scholar、Crossref、arXiv、IEEE 等）、Tavily 通用搜索、Web Operator（Playwright + LLM 操作页面）由 `web_search_executor` 等统一路由与降级。

## 2. 需要配什么 API、加什么依赖、怎么做

### 2.1 环境变量（后端 `development.env` 或等价配置）

| 变量 | 作用 | 不配会怎样 |
|------|------|------------|
| `RA_TAVILY_API_KEY` 或 `TAVILY_API_KEY` | Tavily 网页搜索（`RA_WEB_SEARCH_PROVIDER=tavily` 时的主路径） | 代码会从 `TAVILY` 回退读 `RA_TAVILY`；若都空，通用搜索能力大减，可能走本地占位或仅剩无 Tavily 的学术路由 |
| `RA_SEMANTIC_SCHOLAR_API_KEY` | Semantic Scholar 路由 | 该路由不可用，会走 arXiv / Crossref 等 |
| `RA_IEEE_XPLORE_API_KEY` | IEEE Xplore 路由 | 该路由不可用 |
| `RA_CROSSREF_POLITE_EMAIL` | Crossref 礼貌池（建议填联系邮箱） | 仍可调 Crossref，但建议配置以利稳定性 |
| `RA_WEB_OPERATOR_ENABLED` | 是否启用 Web Operator（默认一般为 true，以代码为准） | 关闭则不会出现 Playwright 路由 |
| `RA_WEB_OPERATOR_MAX_STEPS` 等 | 浏览器步数、超时 | 可选调优 |

学术侧：**arXiv、Crossref 可不配 Key 即用**；Semantic Scholar / IEEE 需 Key 才会进入可选路由。

浏览器侧：目前所有功能未经测试

### 2.2 Python 依赖

- 后端 `requirements.txt` 已包含例如 **`httpx`、`arxiv`、`tavily-python`、`playwright`** 等；部署环境执行：  
  `pip install -r requirements.txt`（注意加了新东西的）

### 2.3 Web Operator（Playwright 浏览器）

- 安装 Python 包后（已执行2.2），在**部署机器**上执行：  
  `playwright install chromium`  
- 若未安装浏览器或 import 失败，**Web Operator 路由不会出现在可用列表**，系统会自动用其他搜索后端。
- 换言之，就算不配这个，顶多影响输出质量，不会因此导致功能失败。

## 3. 带来什么改动

- **报告更可信**：链接与参考文献优先来自真实检索结果，最终报告末尾可看到「参考来源」列表。  
- **单次请求更快、更不易超时**：高负载阶段关闭深度思考；总耗时可能因**轮次增加到 5** 而变长，需结合网关/前端超时一起观察。  
- **检索仍受配置约束**：没配 Tavily/IEEE/S2 等时，会自动降级到其他可用源；Web Operator 为**实验雏形**，复杂站点、验证码、反爬仍可能失败。  
- **更详细说明**：同目录《链路优化》《检索与执行器实现》。
