# 链路优化 (Pipeline Optimization)

## 1. 背景与问题
在早期的 Research Agent 运行中，暴露出以下几个关键问题：
1. **严重幻觉 (Hallucination)**：最终生成的报告中，学术论文的标题、作者和链接经常是 LLM 凭空捏造的，缺乏真实性。
2. **引用丢失**：即使底层搜索工具获取了真实的 URL，这些链接在经过 Reader、Reflector 阶段时被丢弃，导致 Writer 无法在最终报告中提供真实的参考文献。
3. **前端超时**：部分处理节点（特别是迭代反思阶段）由于开启了深度思考（Deep-Thinking），导致 LLM 响应时间过长，触发前端或网关的 HTTP 读超时。
4. **搜索深度不足**：默认的检索与反思轮次较少，无法充分挖掘复杂课题的深度信息。

## 2. 编排逻辑 (Orchestrator) 修正
针对上述问题，对 `orchestrator.py` 进行了核心执行链路的重构：

### 2.1 搜索执行顺序倒置
- **原逻辑**：先调用 Searcher LLM 生成检索策略和“预期”的论文信息，然后再调用实际的搜索工具。这导致 LLM 在没有真实数据的情况下强行编造内容。
- **新逻辑**：**先执行**真实的网络/学术检索（`_search_context`），获取真实的 `citations`。然后将这些真实结果作为 `raw_search_results` 喂给 Searcher LLM。LLM 的角色从“凭空生成”转变为“对真实搜索结果进行清洗、去重和分组”，从根本上消除了搜索阶段的幻觉。

### 2.2 引用链 (Citation Chain) 贯通
为了确保真实的 URL 能够传递到最终报告：
- 在 `_coerce_read_payload_for_pipeline`、`_coerce_reflect_payload` 等数据清洗函数中，增加了对 `references` 字段的提取和继承逻辑。
- 当 Reader 或 Writer 发生 JSON 解析失败触发容错降级（Fallback）时，确保兜底逻辑（`_read_fallback_from_info_groups`、`_fallback_write_payload_from_pipeline`）也能尽可能从上游数据中提取并保留 `references`。
- 在 `_markdown_from_write_json` 中，新增了参考文献的渲染逻辑，在报告末尾自动追加 `## 参考来源` 章节，并使用 Markdown 链接语法 `[1] [标题](URL)` 真实展示。

### 2.3 性能与深度平衡
- **关闭高耗时节点的深度思考**：在 `_llm_call` 中，将 `read`、`reflect`、`write` 阶段的 `enable_thinking` 强制设为 `False`。这大幅降低了单次 LLM 调用的延迟，有效缓解了前端超时问题。
- **提升最大轮次**：修改了 `_max_reflect_rounds` 函数，将默认的最大反思与搜索轮次从 2 轮提升至 5 轮，在保证单轮响应速度的前提下，通过增加迭代次数来加深搜索深度。

## 3. 提示词 (Prompts) 优化
配合 Orchestrator 的链路调整，对 `prompts.py` 中的各角色提示词及其 JSON Schema 进行了严格约束：

- **Searcher (`USER_PROMPT_SEARCH`)**：
  - 引入 `raw_search_results` 变量。
  - 增加硬性约束：“绝对禁止编造不存在的论文或URL。必须从 raw_search_results 中提取真实数据。如果没有真实数据，请仅输出检索方向。”
- **Reader (`USER_PROMPT_READ`)**：
  - JSON Schema 增加 `references` 数组（包含 id, title, url）。
  - 增加约束：“references 必须从 citation_context 提取真实的来源，绝对禁止编造。在 analysis 和 key_points 中必须使用 [1], [2] 等角标引用 references。”
- **Reflector (`USER_PROMPT_REFLECT`)**：
  - 在 `accepted_reader_summary` 的 Schema 中加入 `references`，并要求：“必须原样保留 reader_summary 中的 references”，防止在反思评估时丢失来源。
- **Writer (`USER_PROMPT_WRITE`)**：
  - JSON Schema 增加 `references` 数组。
  - 增加约束：“必须汇总所有子任务的 references 并去重。在 sections 的 content 中，必须使用 Markdown 链接语法 [1](URL) 或 [论文标题](URL) 真实地展示来源，绝对禁止编造任何 URL。”

通过代码编排与提示词的双重优化，系统现在能够稳定、快速地输出包含真实学术引用且无幻觉的高质量研究报告。
