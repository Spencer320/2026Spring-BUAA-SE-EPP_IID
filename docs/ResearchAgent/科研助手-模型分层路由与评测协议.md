# 科研助手模型分层路由与评测协议（P0）

## 1. 文档目标与适用范围
- 目标：基于已给定模型清单，形成可执行的分层路由策略、主备降级机制与统一评测协议，用于支撑科研助手 `P0` 落地。
- 适用范围：用户侧与智能体侧主链路（任务创建 -> 规划 -> 检索 -> 阅读 -> 反思 -> 写作 -> 报告输出）。
- 非目标：不包含管理端页面实现；视频生成能力仅保留扩展位，不纳入本轮启用。

## 2. 固定可用模型清单（输入约束）
- `Qwen3-30B-A3B`
- `DeepSeek-V3.1`
- `Qwen3-235B-A22B`
- `Kimi-K2`
- `BGE-Reranker-V2-M3`
- `DeepSeek-V3.2`
- `GLM-5`
- `Qwen2.5-VL-72B`
- `Qwen_Image`
- `Wan2.2-T2V-A14B`

## 3. 分层路由策略

### 3.1 第一层：主推理与任务编排层（质量优先）
- 目标任务：研究问题拆解、阶段规划、证据整合、反思修正、报告主文生成。
- 模型优先级：
  1. `DeepSeek-V3.2`（主模型）
  2. `Qwen3-235B-A22B`（一级回退）
  3. `Kimi-K2`（二级回退）
- 触发回退条件：
  - 请求超时、服务不可用、连续重试失败；
  - 结构化输出校验失败（例如缺失必要字段）超过阈值；
  - 单轮 token 成本超过预算熔断阈值。

### 3.2 第二层：快速子任务层（速度与稳定优先）
- 目标任务：意图分类、查询改写、字段抽取、结果结构化、简短摘要。
- 模型优先级：
  1. `Qwen3-30B-A3B`（主模型）
  2. `GLM-5`（回退模型）
- 使用约束：
  - 输出必须通过 JSON Schema 校验，不通过则自动重试或回退；
  - 单任务重试上限后交回主推理层进行兜底修复。

### 3.3 第三层：证据重排层（引用质量优先）
- 目标任务：候选证据打分、去噪、引用排序与覆盖率提升。
- 核心模型：`BGE-Reranker-V2-M3`
- 路由规则：
  - 当检索候选数量大于阈值时强制启用重排；
  - 报告写作前执行最终重排，确保高置信来源优先引用。

### 3.4 第四层：多模态理解层（图文输入）
- 目标任务：论文图表理解、图片内容解读、图文混合证据抽取。
- 核心模型：`Qwen2.5-VL-72B`
- 路由规则：
  - 输入中检测到图片或图表引用时启用；
  - 仅当图像对研究结论有增量价值时触发，避免无效调用。

### 3.5 第五层：生成式视觉层（可选能力）
- 图像生成模型：`Qwen_Image`（条件启用，见第 6 节）
- 视频生成模型：`Wan2.2-T2V-A14B`（本轮不启用，仅保留接口扩展位）

## 4. 任务类型到模型的路由映射
- 任务规划/深度推理：`DeepSeek-V3.2 -> Qwen3-235B-A22B -> Kimi-K2`
- 查询改写/分类标注：`Qwen3-30B-A3B -> GLM-5`
- 引用候选重排：`BGE-Reranker-V2-M3`
- 图表解读/图文理解：`Qwen2.5-VL-72B`
- 图示化输出（符合条件时）：`Qwen_Image`
- 视频能力：仅保留配置，不进入执行路径（`Wan2.2-T2V-A14B`）

## 5. 主备模型降级与恢复策略
- 超时降级：主模型超时后按优先级回退，回退链路最多两级，避免无限重试。
- 可用性降级：模型服务返回不可用时直接切换下一优先级，并记录降级事件。
- 质量降级：结构化输出校验连续失败时，切换到更稳定模型并限制生成长度。
- 成本降级：单任务累计 token 达预算阈值后，非关键子任务切换到快速子任务层模型。
- 自动恢复：连续 N 次调用成功后恢复主模型，恢复事件写入审计日志。

## 6. `Qwen_Image` 启用条件与禁用条件

### 6.1 启用条件（同时满足）
- 用户明确请求图示化产出，例如“图示化解释/流程图/研究框架图/可视化摘要”。
- 任务内容不涉及敏感或受限信息，不触发安全策略拦截。
- 文本主报告已形成稳定结论与引用，图像作为补充表达而非替代证据。
- 当前任务未触发成本熔断，且图像生成预算在可控范围内。

### 6.2 禁用条件（任一命中即禁用）
- 用户未显式请求图像产出；
- 内容含敏感信息、版权高风险素材或来源不可追溯；
- 引用覆盖率不足（未达到最低验收阈值）；
- 系统处于降级模式（可用性或成本熔断期间）。

### 6.3 审计与绑定要求
- 图像生成必须绑定同一 `task_id` 与 `step_id`，不得脱离主任务链路。
- 图像结果需关联文本结论段与来源引用清单，保证可追溯。
- 审计字段至少包含：模型名、触发原因、输入摘要、输出摘要、耗时、token/计费信息、安全策略判定结果。

## 7. 模型评测协议

### 7.1 评测目标
- 形成主备推荐：确定各层主模型、回退模型与触发阈值。
- 量化稳定性：验证在超时、服务波动与异常输入下的可靠性。
- 控制成本：在满足质量前提下优化平均耗时与 token 消耗。

### 7.2 评测数据集
- 真实科研问题样本（覆盖不同学科与难度等级）。
- 现有样本集：`docs/test_papers`。
- 评测分桶：
  - 常规文本任务；
  - 高检索密度任务（多来源证据）；
  - 图文混合任务（含图表理解）；
  - 图示化请求任务（验证 `Qwen_Image` 触发规则）。

### 7.3 核心评测指标
- 任务拆解准确率：子问题划分是否完整、去重、可执行。
- 工具调用成功率：检索/阅读/重排/写作链路一次成功比例。
- 引用溯源完整率：结论是否可追溯到有效来源 URL 与步骤记录。
- 平均耗时：端到端任务总耗时与关键阶段耗时。
- token 成本：单任务总消耗与各阶段消耗分布。
- 图像触发准确率：应触发时触发、不应触发时不触发。

### 7.4 评测执行流程
1. 固定数据集与提示词版本，避免评测漂移。
2. 按层执行单模型基线评测，记录质量、稳定性与成本。
3. 执行路由策略联调评测，验证主备切换与回退是否生效。
4. 注入故障场景（超时/服务不可用/输出不合规）验证降级策略。
5. 输出结论与参数建议（优先级、阈值、预算、重试上限）。

### 7.5 结论输出模板（统一格式）
- 分层推荐：每一层给出主模型、一级回退、二级回退（如适用）。
- 阈值建议：超时阈值、重试次数、预算阈值、结构化校验失败阈值。
- 风险说明：已知不稳定场景、易失败输入类型、临时规避策略。
- 发布建议：灰度范围、观察指标、回滚触发条件。

## 8. P0 默认配置建议（可直接转实现）
- 主推理层默认：`DeepSeek-V3.2`，超时后回退 `Qwen3-235B-A22B`，再回退 `Kimi-K2`。
- 快速子任务层默认：`Qwen3-30B-A3B`，失败回退 `GLM-5`。
- 重排层默认：`BGE-Reranker-V2-M3` 常驻启用（候选证据超过阈值时强制）。
- 图文理解层默认：检测到图片输入时启用 `Qwen2.5-VL-72B`。
- 图像生成默认关闭，仅在第 6 节启用条件满足时调用 `Qwen_Image`。
- 视频生成保持关闭，不进入运行时路径。

## 9. 与验收的对齐关系
- 对齐“可运行”：分层路由与降级策略可直接转为执行引擎路由规则。
- 对齐“可审计”：模型选择、切换、失败与图像触发均有事件可追溯。
- 对齐“可管控”：通过阈值、预算、白名单条件控制风险与成本。
