## 1. 这轮更新干了什么

- **运行实体拆成三类**：会话日常走 `BasicOrchestratorRun`；独立深度研究走 `AgentTask`（六阶段流水线）；工作区多轮 Agent 走 `WorkspaceAgentRun`，且**只**在 Basic 编排的 **`agent` 子步**里由 `run_workspace_delegate` **同步**拉起（不与深度研究流水线混成一条链）。
- **HTTP 与产品边界**：科研助手统一挂在 **`/api/research-agent/`**；日常对话/检索/工作区委托与 **`POST .../tasks/deep-research/`** 深度研究入口**已分离**；同一会话并发进行中任务会 **409**（与另一套 `/api/deep-research/` 业务接口区分）。
- **Basic 内规划**：首轮用 **Smart Planner** 产出有序子步（`chat` → `search` → `agent` 等）；从第二步起用 **step_refill** 补全空缺的 `query` / `delegate_prompt` / `prompt`。
- **论文展示区（Paper shelf）**：检索子步可自动合并引用；用户可把工作区文件加入展示区；深度研究的 **`selected_papers`** 仅能引用**当前会话**展示区已有条目。
- **工作区与前端感知**：用户文件走 **`/api/workspace/files`**；Agent 改盘后轮询父 Basic 任务的 **`workspace_fs_generation`**（在 `.../status/` 或会话 `active_task` 上）变化后再拉文件树刷新。
