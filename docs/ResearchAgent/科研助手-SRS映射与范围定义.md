# 科研助手 SRS 逐条映射与完整范围定义（用户侧与智能体侧）

## 1. 文档目标与边界
- 目标：将“科研智能助手 + Deep Research”从当前 Mock 演示态，收敛为可运行、可验收的用户侧与智能体侧交付清单。
- 本文仅覆盖：`FR-AIMK-0004`、`FR-KYZS-0001` ~ `FR-KYZS-0007`。
- 本文不覆盖：管理端页面、管理端流程与管理端验收；仅保留后端事件字段与接口兼容约束。
- 口径原则：以“用户可感知行为 + 智能体可审计行为”为验收标准，不以内部实现细节作为唯一验收依据。

## 2. 术语与验收口径
- 用户侧：用户通过前端发起研究任务、查看过程、进行干预、获取报告与导出结果的全链路体验。
- 智能体侧：任务编排、模型路由、工具调用、审计记录、异常恢复与安全控制。
- 验收通过：满足功能正确性、状态一致性、可追踪性与风险可控性四个维度。

## 4. 完整范围定义（本轮必交付）

### 4.1 核心交付项
- 真实任务创建：替换 Mock 创建，完成参数校验、任务初始化、限频与配额检查。
- 主循环闭环：实现“规划 -> 检索 -> 阅读 -> 反思 -> 写作”状态机，阶段切换与步骤落库完整。
- 实时状态流：用户可持续看到任务阶段、进度、步骤摘要与增量事件。
- 人工干预：支持用户侧允许/终止/修订三类干预动作，具备挂起与恢复机制。
- 报告产出与下载：输出 Markdown/结构化JSON 报告，具备引用绑定与下载能力。
- 反思闭环刚性约束：
  - `reflect` 阶段输出必须为严格 JSON，且包含 `needs_optimization`（`yes|no`）、`suggestions`（string[]）、`reason`（string）；
  - 引擎需支持 `max_reflect_rounds`（最大反思轮数）并强制上限，防止无限循环；
  - 当 `needs_optimization=yes` 时，`suggestions` 必须在下一轮 `plan/search/read` 作为输入上下文，禁止“假优化”（只给 yes 不给可执行修改建议）。

### 4.2 联网搜索能力（新增并纳入必交付）
- 支持“检索 -> 阅读 -> 再检索”迭代式外部信息获取，并与本地RAG证据融合。
- 搜索结果统一结构化字段：`query`、`source`、`published_at`、`confidence`、`url`、`snippet`。
- 增加学术站点友好访问策略：节流、随机抖动、退避重试、来源轮换，避免被判定为异常抓取行为。
- 对外部站点交互失败（限流、验证码、拒绝访问）需降级到可用替代源，并输出可解释失败原因。

### 4.3 本地操作能力（新增并纳入必交付）
- 本地操作采用模板化任务执行，不允许自由拼接 shell；仅允许白名单动作。
- 执行环境受控：工作目录隔离、参数校验、超时终止、输出截断、资源配额限制。
- 高风险本地动作默认进入人工确认，未确认不得执行。
- 本地操作结果必须与任务步骤和引用来源绑定，纳入统一审计事件流。

### 4.4 对外交互与本地操作的统一方法（类 OpenClaw）
- 统一采用“感知 -> 规划 -> 执行 -> 观察 -> 纠偏”闭环，所有动作显式记录前置条件与预期产出。
- 对外部交互与本地操作统一使用“动作卡片”协议：动作类型、风险等级、参数快照、结果摘要、可回放日志。
- 工具调用失败时按策略自动重试/降级，不满足安全策略时直接拒绝并给出可解释原因。
- 审计优先：每次动作均写入可追踪事件，保证用户侧可见状态与智能体侧内部状态一致。

## 5. 用户侧与智能体侧验收用例清单（仅本轮边界）

### 5.1 正常路径
1. 用户提交研究问题并创建任务成功，返回唯一任务ID。
2. 任务进入运行态并按阶段推进，事件流持续输出。
3. 任务完成后报告可查看、可下载，且引用来源完整可读。
4. 用户基于当前结果发起追问，产生新一轮步骤与增量结果。
5. 当 `reflect` 返回 `yes` 时，任务进入下一轮“重规划-再检索-再阅读”；当返回 `no` 或达到 `max_reflect_rounds` 时，进入写作阶段。

### 5.2 异常与风险路径
1. 超时：模型或工具超时后按策略重试；超限后任务失败并带错误原因。
2. 越权：非任务归属用户访问/操作任务被拒绝。
3. 域名阻断：不在白名单的外部链接请求被拒绝并审计留痕。
4. 命令拒绝：不符合命令模板白名单的执行请求被拒绝并反馈原因。
5. 人工中断：用户主动终止后任务进入终态，后续不再继续执行步骤。
6. 人工修订恢复：用户对高风险动作进行修订确认后任务可恢复执行。
7. 学术站点反爬/限流：触发异常行为判定后，系统进入节流+退避并切换替代来源，任务不中断或可解释失败。
8. 本地操作风险拦截：高风险动作未获确认时被阻断，任务进入待确认态并可恢复。

## 6. 不纳入本轮验收的内容
- 管理端页面与管理端完整业务流程。
- 视频生成能力（仅保留扩展位，不在本轮启用）。
- 跨租户/跨组织的高级治理策略。（全部开发完成后可能引入）
