# 科研助手用户侧验收、压测与灰度回滚说明

本文用于执行 `impl-step-6-tests`：覆盖用户侧验收测试、压测、灰度发布与回滚验证。

## 1. 用户侧验收测试

后端新增测试文件：

- `EPP-Backend-Dev/business/tests/test_deep_research_user_acceptance.py`

覆盖范围：

- 用户鉴权（未登录访问拦截）
- 任务状态轮询与增量事件查询
- 报告读取规则（未完成不可读、完成可读）
- 管理端屏蔽后用户报告访问阻断
- 用户主动中止（running 可中止、终态不可中止）
- 跨用户数据隔离
- `follow-up` / `export` 当前 stub 行为校验

执行命令：

```bash
cd /home/Spencer/projects/SE_project/EPP-Backend-Dev
.venv/bin/python manage.py test business.tests.test_deep_research_user_acceptance -v 2
```

验收建议阈值：

- 通过率：100%
- 关键接口：`status/events/report/abort` 不允许回归

## 2. 压测（用户侧轮询链路）

脚本：

- `EPP-Backend-Dev/scripts/deep_research_load_test.py`

示例：

```bash
cd /home/Spencer/projects/SE_project/EPP-Backend-Dev
python scripts/deep_research_load_test.py \
  --base-url http://127.0.0.1:8000 \
  --username <用户名> \
  --password <密码> \
  --task-id <可访问任务ID> \
  --requests 500 \
  --concurrency 50
```

输出指标：

- 成功率、QPS
- 时延：min/avg/p50/p95/p99/max

建议灰度门槛（可按环境调整）：

- 成功率 >= 99.5%
- p95 <= 800ms
- p99 <= 1200ms

## 3. 灰度发布验证

脚本：

- `EPP-Backend-Dev/scripts/deep_research_canary_rollback_check.py`

执行前准备：

1. 获取用户 JWT（可通过 `/api/login`）
2. 准备一个该用户可访问的 `task_id`

执行：

```bash
cd /home/Spencer/projects/SE_project/EPP-Backend-Dev
python scripts/deep_research_canary_rollback_check.py \
  --base-url http://127.0.0.1:8000 \
  --user-token <JWT> \
  --task-id <可访问任务ID> \
  --expect-create-status 400
```

说明：

- 目前用户侧创建任务接口仍为 stub，默认校验 `create` 返回 `400`。
- 当创建能力正式落地后，将 `--expect-create-status` 改为新契约状态码（如 200/201）即可复用同脚本。

## 4. 回滚验证

回滚后重复执行以下两项：

1. `deep_research_canary_rollback_check.py`
2. `deep_research_load_test.py`（至少 200 请求）

通过条件：

- 契约检查 `all_passed=true`
- 关键指标未出现灾难性退化（建议：相对灰度前基线退化不超过 20%）

## 5. 推荐发布节奏（用户侧）

1. 小流量灰度（5%）+ 契约检查
2. 扩容灰度（20%）+ 压测抽检
3. 全量前复核（50%/100% 分段）
4. 若异常，立即回滚并按“回滚验证”复测
